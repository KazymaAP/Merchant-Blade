### ✅ Реализованные системы:

## 2.1 Расширение EventSystem
- **ГОЛОД**: ×300% на еду (вместо ×1.5) - квест доставки 50 зерна
- **ПОЖАР**: уничтожает товар на складе, стройматериалы ×500%
- **СВАДЬБА ЛОРДА**: вино/мясо/ткани ×1.5, налог 0%
- **Предвестники**: слухи за 3-5 дней до события
- Сигнал: event_warning эмитится при приближении события

## 2.2 Динамическая генерация квестов
- `generate_event_quest()` - создаёт квесты для событий
  - При голоде: "Помощь голодающим" (50 зерна за 5 дней, 3000 медяков + +30 репутация)
  - При пожаре: "Восстановление после пожара" (30 стройматериалов за 4 дня, 2500 медяков + +20 репутация)
- `donate_to_event()` - пожертвование товара (+1 репутация за единицу)
- `update_daily()` - проверка дедлайнов квестов
- Интегрировано в game_manager.travel_to_city()

## 2.3 UI для квестов
- QuestPanel отображает:
  - Название и тип квеста
  - Описание и прогресс
  - Дней осталось до дедлайна
- Автоматически обновляется при смене дня
- Сигналы: quest_available, quest_completed

### 📦 Структура события:
```gdscript
{
    "id": "event_id",
    "name": "Название события",
    "price_modifier": 3.0,
    "affected_goods": [1, 2, 3],
    "duration": 7,
    "rumor_warning_days": 5,  # дней до события генерируется предвестник
    "quest_generated": true    # генерируется ли квест
}
```

### 🔄 Интеграция в цикл дня:

EventSystem.update_day() вызывается каждый день:
1. Проверяются предвестники - эмитится signal event_warning
2. Слух добавляется в RumorSystem
3. Квест генерируется в QuestSystem
4. Активные события обновляются

QuestSystem.update_daily() вызывается каждый день:
1. Проверяются дедлайны квестов
2. Обновляется прогресс на основе инвентаря
3. Провалившиеся квесты удаляются, репутация -5
