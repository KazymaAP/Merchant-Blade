# РЕЗЮМЕ ПЕРВОГО ПРОТОТИПА
# Merchant & Blade: Roads of Fortune v0.1.0

## 📦 Что создано

### ✅ Завершённые компоненты:

**Автозагрузки (глобальные системы):**
1. `game_manager.gd` — управление состоянием игрока
   - Капитал (медь/серебро/золото)
   - Текущий день и город
   - Инвентарь с отслеживанием порчи
   - Репутация
   - Сигналы для обновления UI

2. `economy_manager.gd` — экономическая система
   - Загрузка товаров и городов из JSON
   - Динамические цены с модификаторами
   - Методы buy_good() и sell_good()
   - Применение налогов (5%)
   - Обновление цен по дням

3. `save_load.gd` — система сохранений
   - Сохранение в user://savegame.json
   - Загрузка состояния игры
   - Удаление сохранений

**UI Скрипты:**
4. `main_ui.gd` — управление вкладками и главным окном
   - Вкладки (7 штук)
   - Кнопки Сохранить/Загрузить/Выход
   - Заголовок со статусом

5. `trade_panel.gd` — вкладка торговли
   - Список товаров с ценами
   - Кнопки Купить/Продать
   - Поля ввода для количества
   - Отображение кошелька

6. `inventory_panel.gd` — вкладка инвентаря
   - Список товаров
   - Дни порчи для продовольствия
   - Индикатор "осталось дней до порчи"

7. `map_panel.gd` — вкладка карты (путешествия)
   - Три кнопки городов
   - Система путешествий (1 день = 1 переезд)
   - Обновление цен при смене дня

8. `team_panel.gd` — вкладка отряда (заглушка)
   - Информация об игроке и наёмнике
   - Готова для расширения боевой системой

**Сцены:**
9. `scenes/world.tscn` — главная сцена в текстовом формате
   - TabContainer со 7 вкладками
   - Все UI элементы (лейблы, кнопки, полосы прокрутки)
   - Связь со всеми скриптами UI

**Данные (JSON):**
10. `data/goods.json` — товары (5 типов)
    - Зерно (портится через 14 дней)
    - Шерсть, железо, дерево, вино (не портятся)
    - Базовые цены для 3 городов

11. `data/cities.json` — города (3 штуки)
    - Город А, Б, В
    - Налоговые ставки

**Конфигурация:**
12. `project.godot` — обновлен с:
    - Стартовая сцена: res://scenes/world.tscn
    - Три автозагрузки
    - Версия: 0.1.0

**Документация:**
13. `README.md` — полное описание прототипа
14. `DEVELOPMENT_GUIDE.md` — гайд по расширению

## 🎮 Core Loop (работающий прототип)

1. **Стартуете в Городе А** с 5000 медяков (50 серебра)
2. **Смотрите цены** товаров в текущем городе
3. **Покупаете** (например, 50 единиц зерна за 600 медяков)
4. **Путешествуете** в Город Б (вкладка "Карта")
5. **Продаёте** зерно (цена выше, но с налогом 5%)
6. **Смотрите инвентарь** (зерно портится через 14 дней)
7. **Повторяете** для прибыли

## 🔑 Ключевые особенности

✅ **Три валюты** с автоконвертацией
✅ **Динамические цены** (базовая ± 10%, + модификаторы)
✅ **Портящиеся товары** (зерно исчезает на день 14+)
✅ **Налоги на продажу** (5% от дохода)
✅ **Система путешествий** (1 день в пути)
✅ **Сохранение/загрузка** (полное состояние игры)
✅ **Модульная архитектура** (легко расширять)
✅ **Data-driven система** (JSON для товаров и городов)

## 📋 Сигналы и события

### GameManager сигналы:
- `money_updated(copper: int)` — изменение капитала
- `day_updated(day: int)` — смена дня
- `city_changed(city_index: int, city_name: String)` — смена города
- `inventory_updated` — изменение инвентаря

### EconomyManager сигналы:
- `price_updated` — обновление цен
- `buy_completed(good_id, quantity, total_cost)`
- `sell_completed(good_id, quantity, total_revenue)`

## 💡 Советы по игре

1. **Лучший профит** — купить в дешёвом городе, продать в дорогом
2. **Арбитраж** — зерно дешевле в А (12м), дороже в Б (18м)
3. **Осторожно с едой** — зерно портится через 14 дней!
4. **Налоги** — каждая продажа -5%, планируйте это
5. **Большие продажи** — если продали 10+, цена упадёт на 3 дня

## 📚 Для разработчиков

Все компоненты независимы и тестируемы:
- EconomyManager работает без UI
- GameManager хранит только состояние
- UI скрипты подключаются к сигналам

Код готов для CI/CD тестирования:
- Нет жёсткой связи между системами
- JSON легко подменять для тестов
- Все важные методы логируют действия
