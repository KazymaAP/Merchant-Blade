# ИНСТРУКЦИИ ПО ИСПОЛЬЗОВАНИЮ ПРОТОТИПА

## ✅ Что уже реализовано

### 1. Автозагрузки (доступны везде через имена)
- **GameManager** — состояние игрока (деньги, инвентарь, день, город, репутация)
- **EconomyManager** — экономика (товары, цены, куплю/продажу)
- **SaveLoad** — сохранение/загрузка в JSON

### 2. Система торговли
✅ Покупка товаров в любом городе
✅ Продажа товаров (применяется налог 5%)
✅ Динамические цены (базовая цена ± 10% + модификаторы)
✅ Модификаторы цен при больших продажах (10+ единиц = -10% на 3 дня)

### 3. Инвентарь
✅ Отслеживание всех товаров
✅ День порчи для продовольствия (зерно портится на день 14+)
✅ Автоудаление испорченных товаров при смене дня

### 4. Управление деньгами
✅ Три валюты (медь/серебро/золото) с автоконвертацией
✅ Вся математика в медяках (целые числа, без float)
✅ Форматированное отображение: "50 з. 30 с. 45 м."

### 5. Путешествия
✅ Переезд между 3 городами (Город А, Б, В)
✅ Каждый переезд = +1 день
✅ При смене дня обновляются цены и проверяется порча

### 6. Интерфейс
✅ Вкладки: Торговля, Карта, Инвентарь, Отряд
✅ Заглушки для: Репутация, Квесты, Таверна
✅ Кнопки сохранения/загрузки
✅ Заголовок с информацией о дне, городе и капитале

### 7. Сохранение/загрузка
✅ JSON в user://savegame.json
✅ Сохраняет: капитал, инвентарь (с датами порчи), день, город, репутацию

## 🎮 Как тестировать

### Сценарий 1: Арбитраж
1. Запустите игру (вкладка "Торговля")
2. Купите зерна (дешево в Городе А: 12 м.)
3. Перейдите в Город Б (вкладка "Карта")
4. Продайте зерно (дороже здесь, но с налогом)
5. Повторите с другими товарами

### Сценарий 2: Порча продовольствия
1. Купите 100 единиц зерна (1200 медяков)
2. Не продавайте 14 дней
3. На день 14+ просмотрите инвентарь
4. Зерно должно исчезнуть

### Сценарий 3: Модификаторы цен
1. Купите 15 единиц любого товара
2. Продайте все 15 сразу
3. Вернитесь в этот город на следующий день
4. Цена упадёт на ~10%

## 🔧 Добавление новых товаров (легко!)

**Шаг 1:** Добавьте в `data/goods.json`:
```json
{
  "id": 6,
  "name": "Специи",
  "category": "food",
  "basePrice": [200, 180, 220],
  "spoilDays": 7
}
```

**Шаг 2:** Готово! EconomyManager автоматически загрузит и покажет в торговле.

---

## 🗂️ Добавление новых городов

**Шаг 1:** Обновите `data/cities.json`:
```json
{
  "id": 3,
  "name": "Город Г",
  "tax_rate": 0.05
}
```

**Шаг 2:** Добавьте цену для каждого товара в `basePrice` в `goods.json`:
```json
"basePrice": [12, 18, 15, 20]  // добавляем 4-й элемент
```

**Шаг 3:** В `map_panel.gd` города загружаются автоматически из JSON!

## 📊 Архитектура данных

### GameManager.inventory
```gdscript
{
  1: { "quantity": 50, "spoil_day": 15 },  # зерно, портится в день 15
  2: { "quantity": 10 }                     # шерсть, не портится
}
```

### EconomyManager.current_prices
```gdscript
{
  0: { 1: 12, 2: 35, 3: 50 },  # Город А
  1: { 1: 18, 2: 28, 3: 55 },  # Город Б
  2: { 1: 15, 2: 40, 3: 45 }   # Город В
}
```

### SaveLoad (user://savegame.json)
```json
{
  "player": {
	"current_city_index": 1,
	"money_in_copper": 12500,
	"current_day": 5,
	"reputation": 55
  },
  "inventory": {
	"1": { "quantity": 20, "spoil_day": 19 },
	"2": { "quantity": 5 }
  }
}
```

## 🚀 Как расширять функциональность

### Добавить боевую систему
1. Создайте `scripts/battle/battle_controller.gd`
2. Создайте `scripts/battle/battle_grid.gd`
3. Создайте `scenes/battle.tscn`
4. Вызывайте из MapPanel при путешествии (сейчас это заглушка)

### Добавить конкурентов
1. Создайте `scripts/entities/competitor.gd`
2. Добавьте в EconomyManager список конкурентов
3. Конкуренты должны влиять на цены (снижать предложение)

### Добавить квесты
1. Создайте `scripts/quest_system.gd` (автозагрузка)
2. Добавьте данные квестов в `data/quests.json`
3. Создайте `scripts/ui/quest_panel.gd`

### Добавить события (голод, ярмарка)
1. Создайте `scripts/events.gd` (автозагрузка)
2. Добавьте данные событий в `data/events.json`
3. События должны изменять цены через EconomyManager

## ✨ Советы по дизайну

### Экономический баланс
- Базовая цена товара должна учитывать расстояние между городами
- Налог на продажу (5%) — компромисс между доходом и затратами
- Модификаторы цен должны стимулировать торговлю между городами

### Как сделать интересным
- Различные сезоны (скажем, летом зерно дешевле)
- События, которые влияют на цены (голод → зерно дорожает)
- Конкуренты, которые покупают товары (создают спрос)
- Долги и проценты (если капитал падает)

### Баланс сложности
- Сложность: количество конкурентов и частота событий
- Стартовый капитал: сейчас 5000 медяков (50 серебра)
- Налоги: сейчас 5%, можно сделать зависимыми от города
